# README todolist
## Добавленные мной сущности

- # Токен
- ID токена
- ID владельца токена
- Сам токен


- # Проект
- ID проекта
- Название проекта
- Создатель проекта


- # Связь проекта и задачи
- ID проекта
- ID задачи

## Авторизация и регистрация
 - Через JWT
 - В localstorage хранится токен, который крепится в заголовок каждого запроса
 - При каждом обновлении страницы access token отправляется и проверяется

## Добавление сотрудника
 - Задачи можно выдавать только своему сотруднику. Чтобы добавить себе сотрудника, нужно выбрать из списка всех доступных нужного человека. 
 - В списке доступны все пользователи, которые еще не являются сотрудниками другого пользователя

## Создание проекта
 - Все задачи хранятся внутри одного проекта
 - Создание проекта происходит через соответствующую кнопку, для создания нужно ввести только название


## Создание задачи
 - Задачу может создать только владелец проекта
 - Задача при создании имеет статус "К выполнению"

## Изменение статуса задачи
 - Изменить задачу могут только ответственный за задачу и создатель проекта
 - При изменении статуса задача автоматически уходит в новую соответствующую колонку

## Мои задачи
 - В моих задачах хранятся все задачи, где пользователь ответственный
 - При клике на задачу происходит переход к проекту, где эта задача находится


## Фильтрация
 - Фильтровать по дате - все задачи выстраиваются по дате завершения от самой скорой до самой дальней
 - Фильтровать по пользователю(доступно только владельцу задачи) - фильтрация по ФИО, введенным в соответствующем поле. 
 - Чтобы сбросить фильтр, нужно нажать "Найти" с пустой строкой :)
